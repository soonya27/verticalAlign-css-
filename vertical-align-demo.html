vertical-align 시각 가이드

inline / inline-block 요소가 한 줄 안에서 어떻게 정렬되는지 눈으로 확인하는 페이지.
각 데모의 위 실선과 아래 실선은 line box의 경계.

1. 한 줄에 모두 비교

같은 텍스트 라인에 48×40 박스를 6개 넣고 각각 다른 vertical-align 값을 줬어. 어디로 가는지 한눈에 비교.

line box top line box bottom Ag텍스트 base top btm mid tTop tBtm Ag텍스트
baseline (기본) top bottom middle text-top text-bottom

2. 각 값 자세히 보기

baseline (기본값)

박스의 아래쪽이 글자의 baseline에 앉음. 영문 알파벳이 앉는 그 선.

Ag텍스트 base Ag텍스트

top

박스의 top이 line box의 top에 붙음. 폰트 메트릭과 무관하게 line box 자체에 정렬.

Ag텍스트 top Ag텍스트

bottom

박스의 bottom이 line box의 bottom에 붙음.

Ag텍스트 btm Ag텍스트

middle ⚠️ 함정

박스의 세로 중심을 baseline + (x-height / 2) 에 맞춤.
line box 한가운데가 아님. x-height는 소문자 x 높이 기준이라, 한글이나 큰 글자에서는 시각적 중심과 어긋나 보이는 일이 잦아.

Ag텍스트 mid Ag텍스트

text-top / text-bottom

박스의 top/bottom을 부모 폰트의 ascent / descent line에 맞춤. line box보다 좁은 "글자가 실제로 그려지는 영역"의 경계.

Ag텍스트 tTop tBtm Ag텍스트

3. line-height에 따라 어떻게 달라지는가

같은 폰트 크기라도 line-height가 커지면 line box가 커져.
top / bottom 은 line box 끝에 묶이므로 line-height에 따라 위치가 변하지만, baseline / text-top / text-bottom 은 폰트 자체에 묶여있어서 거의 변하지 않아.

line-height: 40px Ag top mid btm tTop Ag
line-height: 120px Ag top mid btm tTop Ag

4. 프로젝트에서 쓴 트릭

필수 표시 아이콘을 폰트 메트릭에 의존하지 않고 라인 중앙에 정확히 두는 방법.
① 아이콘 박스의 height를 line-height와 동일하게 → ② vertical-align: top 으로 line box 전체를 점유 → ③ 그 안에서 background-position: center 로 실제 아이콘은 중앙 정렬.

middle 설문조사

↑ vertical-align: middle + 16×16 박스 → 한글에서 살짝 위로 떠 보임

fix 설문조사

↑ 16×24 박스 (점선) + vertical-align: top + 안쪽에 16×16 아이콘을 background center → 정확히 중앙

/* 핵심 코드 */
.question-label {
  line-height: 24px;
}
.question-label::after {
  width: 16px;
  height: 24px;             /* line-height와 동일 */
  vertical-align: top;      /* line box 전체 점유 */
  background: url(...) no-repeat center / 16px 16px;
}
왜 이게 더 정확한가?
middle은 폰트의 x-height에 묶여있어서 한글 폰트에서는 시각 중심과 어긋남. 박스를 line box와 같은 크기로 키우면 어떤 폰트를 써도 항상 기하학적 중앙에 위치.
